# F1 — summary miscounts decisions in flat mode

## Problem
`spectacular summary` reported `Decisions: 0` on workspaces using flat DECISIONS.md, despite ADR prose blocks being present. Data was intact; the count lied.

## Intended behavior
`summary` should report the true count of decisions regardless of storage mode — flat (prose blocks in DECISIONS.md) or index (per-file D<N>.md).

## Root cause
The count walked `_iter_md decisions`, which globs `.spectacular/decisions/*.md`. That folder only exists in **index** mode. In **flat** mode the ADRs are `## <date> — <title>` prose blocks in DECISIONS.md, so the glob matched nothing → count 0.

## Fix
`cli/spectacular` ~line 4002: mode-detect the count. If `.spectacular/decisions/` exists, count files via `_iter_md`; else if DECISIONS.md exists, `grep -c '^## '` the prose blocks (guarded `|| true` for bash-3.2 empty-match exit 1).

## Success criteria
A flat-mode workspace with N ADR prose blocks reports `Decisions: N`; index mode still counts per-file entries correctly.

## Verified by
`tests/cli/decide.test.sh` → `scenario_flat` asserts `summary` reports `Decisions:  2` after two flat `decide` calls; `scenario_index` confirms index mode still counts correctly. 4/4 pass.

## Signature
`summary` reports 0 for a collection that has entries · a count via `_iter_md <coll>` when the collection has a flat single-file mode · glob over a mode-specific folder that may not exist.
