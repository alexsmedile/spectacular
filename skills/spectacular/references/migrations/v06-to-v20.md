# v0.6 → v2.0 — OKF Restructure and Pluralization

## Detection

- `workspace_schema` in `.spectacular/config.yaml` is `"0.6"` or lower.
- Root index files like `DECISIONS.md`, `MEMORY.md`, `SESSIONS.md`, `ROADMAP.md`, `SPEC.md` exist at `.spectacular/` root instead of inside their respective subdirectories.
- Singular subdirectories like `memory/`, `roadmap/`, `feedback/`, `audit/`, `debug/` are found in the workspace.

## Steps

1. **Rename Directories to Plural:**
   - `.spectacular/memory/` ➔ `.spectacular/memories/`
   - `.spectacular/roadmap/` ➔ `.spectacular/roadmaps/`
   - `.spectacular/feedback/` ➔ `.spectacular/feedbacks/`
   - `.spectacular/audit/` ➔ `.spectacular/audits/`
   - `.spectacular/debug/` ➔ `.spectacular/debugs/`
   - `.spectacular/requests/*/feedback/` ➔ `.spectacular/requests/*/feedbacks/`

2. **Relocate Index Files:**
   - `.spectacular/DECISIONS.md` ➔ `.spectacular/decisions/index.md`
   - `.spectacular/MEMORY.md` ➔ `.spectacular/memories/index.md`
   - `.spectacular/SESSIONS.md` ➔ `.spectacular/sessions/index.md`
   - `.spectacular/ROADMAP.md` ➔ `.spectacular/roadmaps/index.md`
   - `.spectacular/SPEC.md` ➔ `.spectacular/specs/index.md`

3. **Consolidate & Flatten specifications:**
   - For any nested capability SPEC files under `.spectacular/specs/<name>/SPEC.md`, rename them to `.spectacular/specs/<name>.md`.
   - Remove the empty capability subfolders.

4. **Rename/Prefix existing files in `decisions/` and `memories/`:**
   - Rename decisions files to `decisions/D<N>-<slug>.md` where `<slug>` is derived from the H1 title in the file.
   - Prefix memories files sequentially with `M<N>-<slug>.md`.

5. **Rewrite Internal Markdown Links:**
   - Update all references to root `.md` indices to point to their relocated subfolder `index.md` targets.
   - Pluralize directory links (e.g. `memory/` ➔ `memories/`, `feedback/` ➔ `feedbacks/`).
   - Rewrite capability folder links `specs/<cap>/SPEC.md` to `specs/<cap>.md`.
   - Rewrite decision/memory links to their prefixed targets.

## Rollback

**Not reversible** in the registry sense. Workspace structures can be manually rolled back, but due to link rewriting, automated downgrade is not supported.

## Validation

- All index files are located inside folders as `index.md`.
- No nested folders exist under `specs/`.
- No singular directories are present under `.spectacular/`.
- Running `spectacular doctor` reports no errors or warnings.
- `workspace_schema` in `.spectacular/config.yaml` is `"2.0"`.
