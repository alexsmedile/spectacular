# Fix — flat-mode `decide` returns exit 1 despite success

## Bug
`spectacular decide "..."` printed `✓ appended decision` and *did* persist to DECISIONS.md, but returned exit code 1 — reading as "reported success but didn't persist" to callers checking `$?`. The user-visible signal of the bug.

## Root cause
The last statement of `cmd_decide`'s flat branch was `[[ -n "$session" ]] && info ...`. With no open session, `$session` is empty → the `[[ ]]` test is false → the `&&` short-circuits → the compound statement returns 1 → the function returns 1. The append had already succeeded; only the exit code lied.

## Fix
`cli/spectacular` ~line 1273: add an explicit `return 0` after the `[[ -n "$session" ]] &&` line so the successful append determines the exit code, not the trailing conditional.

## Verified by
`tests/cli/decide.test.sh` → `scenario_flat` runs `decide` with no open session and asserts exit 0 (was 1). Green.
